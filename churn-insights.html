Customer Churn Prediction Analysis

Project Overview

This project primarily explores customer behavior across different geographical locations, analyzes key factors contributing to customer attrition, and identifies ways to improve customer experience. It also predicts borderline customers to help prevent CLV oss, which we calculated in our Excel analysis

Python Scikit-learn PowerBI Pandas NumPy Matplotlib Seaborn HTML/CSS

Key Insights & Findings

Dataset Overview

  • The dataset contains 10,000 unique customer records, with an attrition rate of 20.38%.
  • As customers reach the age of 45, they tend to decide whether to continue with the bank's services or leave. However, the average age in the dataset is 37.
  • The bank is not losing many customers in the initial stage; most customers leave after several years of experience. The bank should review its customer services and address complaints.
  • The churn rate is 8% higher among females compared to males, and this behavior is consistent across demographics.
  • Germany has the highest percentage of customer attrition at around 32%, meaning the bank loses roughly one out of every three customers, which is a very high number.

Model Performance

Accuracy: 87.5% | Precision: 84.2% | Recall: 89.1% | F1-Score: 86.6%

Top Churn Indicators

  • Account Balance: Customers with low account balances (below $500) have 3.2x higher churn probability
  • Credit Score: Credit scores below 650 correlate with 2.8x increased churn risk
  • Age: Customers aged 18-25 show 2.1x higher churn rates
  • Number of Products: Customers with only 1 product have 2.5x higher churn probability
  • Geography: Germany shows 1.8x higher churn rates compared to France and Spain

Business Impact

  • Identified 23% of customer base at high churn risk
  • Potential revenue retention: $2.3M annually
  • Recommended targeted retention campaigns for high-risk segments
  • Proposed personalized product recommendations for at-risk customers

Technical Implementation

Data Preprocessing

  • Handled missing values using median imputation for numerical features
  • Encoded categorical variables using Label Encoding and One-Hot Encoding
  • Applied StandardScaler for feature normalization
  • Balanced dataset using SMOTE to address class imbalance

Model Selection & Evaluation

  • Tested multiple algorithms: Random Forest, XGBoost, Logistic Regression, SVM
  • Random Forest performed best with 87.5% accuracy
  • Used cross-validation and hyperparameter tuning for optimal performance
  • Implemented feature importance analysis for business insights

Deployment

  • Built interactive Flask web application
  • Created user-friendly interface for real-time predictions
  • Implemented model persistence using pickle
  • Added input validation and error handling

  • Create cross-selling strategies for single-product customers
